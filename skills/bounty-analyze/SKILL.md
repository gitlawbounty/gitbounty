# bounty-analyze

Analyze an open bounty and return structured insight: difficulty, required skills, estimated effort, and an "alpha rating" indicating whether the reward is fair, underpriced, or overpriced for the work.

## Inputs

| Field | Type | Description |
|---|---|---|
| `bountyId` | `number` | Numeric ID of the bounty on `GitlawbBounty.sol` |

## Outputs

```jsonc
{
  "bountyId": 42,
  "difficulty": "easy" | "medium" | "hard" | "legendary",
  "skills": ["rust", "typescript", "solidity", "...] ,
  "estimatedEffort": "1-2 hours" | "1-3 days" | "1 week" | "2+ weeks",
  "alphaRating": "underpriced" | "fair" | "overpriced",
  "alphaReason": "free-form string explaining why",
  "pitfalls": ["string", "..."],
  "tldr": "one-sentence summary",
  "generatedAt": "ISO timestamp"
}
```

## Invocation

```bash
# via gitbounty REST API
curl https://gitbounty.app/api/scout/42

# returns the analyze output above
```

## Implementation

The analysis is generated by an LLM (Groq Llama 3.3 70B by default). The bounty's `title`, `repoOwner`, `repoName`, `issueId`, and `amount` are passed as context. The LLM is given the analysis schema as a JSON output target.

Results are cached at the edge for 15 minutes per bounty.

## Example

Bounty `#42` — "fix ci runner deadlock", 500,000 $GITLAWB.

```jsonc
{
  "bountyId": 42,
  "difficulty": "medium",
  "skills": ["rust", "ci", "concurrency"],
  "estimatedEffort": "1-3 days",
  "alphaRating": "fair",
  "alphaReason": "CI deadlock fixes are non-trivial but tractable for an experienced Rust dev. 500k $GITLAWB sits in market range for medium-skilled work.",
  "pitfalls": [
    "deadlocks are often heisenbugs — repro environment matters",
    "verify root cause before patching; symptom fixes regress"
  ],
  "tldr": "Solid medium bounty for a Rust dev with CI/concurrency experience.",
  "generatedAt": "2026-05-18T17:42:00Z"
}
```
